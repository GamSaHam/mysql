

에러 로그파일
MySQL 설정 파일(my.cnf)에서 log_error라는 이름의 파라미터로 정의된 경로에 생성된다.

InnoDB의 경우 MySQL 비정상적으로 종료됐다면 다시 시작되면서 완료되지 못한 트랜잭션을 정하가고 디스크에 기록되지 못한 데이터가 있으면
다시 기록하는 재처리 작업을 하게 된다. 간혹 문제가 있어서 복구되지 못할 때는 해당 에러 메시지를 출력하고 MySQL은 다시 종료
될 것이다. 

쿼리 도중 발생하는 문제점은 사전 예방이 어려우며, 데이터 베이스의 숨겨진 문제점을 해결 하는 데 많이 도움될 것이다.

비정상적으로 종료된 커넥션 메시지(Aborted connection)

InnoDB의 모니터링 또는 상태 조회(SHOW ENGINE INNODB STATUS같은)의 결과 메시지

MySQL 종료 메시지
Received SHUTDOWN from user ... 사용자 서버 종료
종료 관련 메시지가 없거나 스택 트레이스(대표적으로 16진수의 주소값이 잔뜩 출력되는)와 같은 내용은 출력되는 경우에는 비정상적인 종료로
간주하는수 밖에 없다.
